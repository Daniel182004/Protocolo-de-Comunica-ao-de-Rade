|User Datagram Protocol

Em redes de computadores , o User Datagram Protocol ( UDP ) é um dos principais protocolos de comunicação do conjunto de protocolos da Internet usado para enviar mensagens (transportadas como datagramas em pacotes ) para outros hosts em uma rede IP ( Internet Protocol ). Dentro de uma rede IP, o UDP não requer comunicação prévia para configurar canais de comunicação ou caminhos de dados. O UDP usa um modelo simples de comunicação sem conexão com um mínimo de mecanismos de protocolo. O UDP fornece somas de verificação para integridade de dados e números de porta para endereçar diferentes funções na origem e no destino do datagrama. Ele não possui diálogos de handshake e, portanto, expõe o programa do usuário a qualquer falta de confiabilidade da rede subjacente; não há garantia de entrega, pedido ou proteção duplicada. Se forem necessários recursos de correção de erros no nível da interface de rede, um aplicativo pode usar o Transmission Control Protocol (TCP) ou o Stream Control Transmission Protocol (SCTP), projetados para essa finalidade.

Atributos do User_Datagram_Protocol

  • É orientado a transações, adequado para protocolos simples de resposta a consultas, como o Domain Name System ou o Network Time Protocol .

  • Ele fornece datagramas, adequados para modelagem de outros protocolos, como encapsulamento de IP ou chamada de procedimento remoto e o Network File System .

  • É simples, adequado para bootstrap ou outros propósitos sem uma pilha de protocolos completa , como DHCP e Trivial File Transfer Protocol .

  • É stateless , adequado para um grande número de clientes, como em aplicativos de streaming de mídia , como IPTV .

  • A falta de atrasos de retransmissão o torna adequado para aplicações em tempo real, como Voz sobre IP , jogos online e muitos protocolos usando o Real Time Streaming Protocol .

Protocolo de Comunicaçao de Rade

RDP

RDP (Remote Desktop Protocol) é um protocolo para conectar um usuário a uma área de trabalho remota por meio de um servidor de terminal. A tecnologia surgiu na década de 90 e é usada até hoje. O desenvolvimento é totalmente proprietário, é fornecido apenas em sistemas operacionais da família Windows. No entanto, você pode se conectar por meio de outros sistemas operacionais usando aplicativos de terceiros.

  • Conexão de protocolo ao sistema operacional do servidor para configuração ou gerenciamento. Isso ocorre porque o Windows Server oferece suporte a duas conexões remotas e um logon local em paralelo, enquanto as estações de trabalho oferecem suporte a apenas uma opção.
  • O RDP serve como um transporte para se conectar a um servidor de terminal. Este modo está disponível apenas para sistemas operacionais de servidor. O número de conexões é ilimitado, mas você precisará instalar um servidor de licenças.
  • Conexão a uma estação de trabalho. O usuário, usando o RDP, conecta-se ao computador do escritório ou de casa para continuar trabalhando.

Se você deseja estabelecer uma conexão entre o sistema operacional da família Windows e o Linux, para o segundo, use aplicativos que suportem o trabalho com o protocolo RDP, por exemplo, Remmina, Rdesktop, Vinagre, etc.

Princípio da Operação O RDP é baseado no protocolo TCP. A conexão cliente-servidor ocorre na camada de transporte. Após a inicialização, o usuário é autenticado. Em caso de confirmação bem-sucedida, o servidor transfere o controle para o cliente. O protocolo RDP suporta internamente canais virtuais, através do qual funções adicionais do sistema operacional são transferidas para o usuário, por exemplo, você pode imprimir um documento, reproduzir um vídeo ou copiar um arquivo para a área de transferência.

Segurança A tecnologia oferece segurança de conexão de duas maneiras. O primeiro usa um subsistema interno (Standard RDP Security). Um par de chaves RSA e um certificado de chave pública são gerados na máquina remota e assinados por um deles. Durante a conexão, o cliente recebe um certificado assinado e uma chave pública, que é usado para selecionar a tecnologia de criptografia de dados. O comprimento da chave de criptografia depende da versão do sistema operacional; em versões mais antigas do Windows Server 2008, o valor era de 168 bits. A autorização ocorre após a confirmação do certificado e digitação da senha. O segundo método sugere o uso de ferramentas de segurança externas (Enhanced RDP Security). Como exemplo, vamos falar sobre o protocolo TLS. Durante a inicialização da conexão, ele criptografa a conexão e verifica a permissão do usuário para efetuar login. Essa abordagem reduz a carga no servidor de terminal com um grande número de conexões simultâneas. O princípio de operação é semelhante à primeira opção. Via de regra, as grandes empresas usam a segunda opção e as pequenas empresas a primeira. O protocolo RDP é usado para conexão remota apenas no ambiente Windows. A conexão é totalmente segura, porque usa criptografia e procedimento de autenticação do usuário. Para outros sistemas operacionais, existe um software especial que suporta o trabalho com RDP.
